Monster Hunter World API demo

This is an API for the Monster Hunter World video game. It is basically an extension of the in-game hunter manual. Users can look up just about anything that's related to the game, including armor, skills, weapon motion value, decorations, and monster information. For example, https://mhw-db.com/armor/sets request will return all the armor sets in the game, with relavent information such as defence stats, decoration slots and skills.

Here are some of the available commands, useage should be self-explanatory

  • ailments
  • armor
  • armor/sets
  • charms
  • decorations
  • events
  • items
  • monsters
  • motion-values
  • skills
  • weapons

Assuming that every command here represents a class of in-game objects, the query will return all objects in that class. If user wants the query to be more specific, they can specify the item id (stored in each object), which is unique for every element in the same class. For example: https://mhw-db.com/weapons/1 returns the weapon Buster Sword 1 with type great-sword.

For full description of the API, see mhw-db.

Options I used

In this assignment, I used the query API methods given by the website to request for a skill by name https://mhw-db.com/skills?q={"name":"Critical Eye"}, which returns the unique skill json object Critical Eye. For simplicity, only part of that information is displayed.

This method should be applicable to every class mentioned above.

Applications

Obviously this API is only useful for Monster Hunter World (unless some Capcom developer decides to refer back to older games when developing the newest title. But they probably have their own database, possibly using another api).

Inside the game, however, this api is a life saver. Players can search for information that are not presented in the in game manual. Like motion-values which in general dictates how often a move set should be utitlized in gameplay (unless you like low damage numbers in which case you are a weirdo). weapons allows players to see all weapons in the game even before they unlock them. So they can start grinding towards their dream weapon immediatly, using loot drop information found in monsters and quest rewards found in events.

As mentioned above, every item in the game has a unique id. Usually this correspond with how inventory is stored. If you have a way to access the memory location of your inventory, you can add items using their ids even if you never acquired them. This could break your save, you have been warned.
